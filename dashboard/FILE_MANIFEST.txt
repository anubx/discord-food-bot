SRC/APP/API (Backend Routes)
============================
src/app/api/auth/[...nextauth]/route.ts
  - NextAuth.js Discord OAuth handler
  - GET/POST handlers for auth flow

src/app/api/stats/route.ts
  - GET only
  - Returns dashboard data for authenticated user
  - Calculates totals, streaks, charts

src/app/api/meals/route.ts
  - POST only
  - Saves meal to PostgreSQL
  - Parses AI response ($$TOTALS$$)
  - Auto-logs food water

src/app/api/analyze/route.ts
  - POST only
  - Sends food to Google Gemini API
  - Handles photo upload or text input
  - Returns AI analysis with macros

AUTHENTICATION FLOW
===================
1. User visits /login
2. Clicks "Sign in with Discord"
3. Redirects to Discord OAuth
4. Discord confirms, redirects to /api/auth/callback/discord
5. NextAuth creates session with Discord ID in JWT
6. Redirected to dashboard (/)
7. Dashboard checks session, fetches user data from DB using Discord ID

API FLOW
========
Dashboard (/):
1. Checks session (useSession hook)
2. Calls GET /api/stats
3. Returns user targets, today's totals, meals, charts
4. Displays all components with real data

Meal Logging (/log):
1. Photo upload / text description
2. User clicks "Analyze Meal"
3. POST /api/analyze → Gemini
4. AI returns nutrition with $$TOTALS$$
5. User sees parsed nutrition
6. Click "Log Meal"
7. POST /api/meals → saves to DB
8. Redirects to dashboard

DATABASE INTEGRATION
====================
- Uses same PostgreSQL as Discord bot
- Reads from: user_settings, meals, weight_log, water_log
- Writes to: meals, water_log
- Discord ID is primary key across all tables
- 4 AM food day boundary (matches bot logic)
- Connection pooling (max 5 connections)
- SSL disabled for railway.internal

DEPLOYMENT
==========
Ready to deploy to:

- Traditional Node.js server
- Railway, Fly.io, AWS, etc.

Environment variables needed:
- DISCORD_CLIENT_ID
- DISCORD_CLIENT_SECRET
- NEXTAUTH_URL (your domain)
- NEXTAUTH_SECRET (generated)
- DATABASE_URL (PostgreSQL connection)
- GEMINI_API_KEY
