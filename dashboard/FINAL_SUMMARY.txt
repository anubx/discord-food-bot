DATABASE SCHEMA
================================================================================

Required Tables (from Discord bot database):

user_settings
  - user_id (TEXT, PRIMARY KEY) - Discord ID
  - display_name (TEXT)
  - target_kcal (INTEGER)
  - protein_target (INTEGER)
  - carbs_target (INTEGER)
  - fat_target (INTEGER)
  - water_target (INTEGER)
  - is_premium (BOOLEAN)
  - created_at (TIMESTAMP)

meals
  - id (UUID, PRIMARY KEY)
  - user_id (TEXT, FK)
  - day_key (DATE)
  - window_idx (INTEGER) [0-5 meal window]
  - timestamp (TIMESTAMP)
  - description (TEXT)
  - kcal (INTEGER)
  - protein_g (DECIMAL)
  - carbs_g (DECIMAL)
  - fat_g (DECIMAL)
  - water_ml (INTEGER)
  - photo_url (TEXT)
  - raw_analysis (TEXT)
  - created_at (TIMESTAMP)

weight_log
  - id (UUID, PRIMARY KEY)
  - user_id (TEXT, FK)
  - day_key (DATE)
  - weight_kg (DECIMAL)
  - created_at (TIMESTAMP)

water_log
  - id (UUID, PRIMARY KEY)
  - user_id (TEXT, FK)
  - day_key (DATE)
  - amount_ml (INTEGER)
  - source (TEXT) ['manual' or 'food']
  - timestamp (TIMESTAMP)
  - created_at (TIMESTAMP)

================================================================================
KEY IMPLEMENTATION DETAILS
================================================================================

Food Day Boundary (4 AM):
  - Food logged 12:00 AM - 3:59 AM → counts toward YESTERDAY
  - Food logged 4:00 AM - 11:59 PM → counts toward TODAY
  - Implemented in: /api/stats and /api/meals routes
  - Matches Discord bot logic

Meal Windows (Auto-categorized):
  - 7:00-9:59 AM → Breakfast (🌅)
  - 10:00-12:59 PM → Morning Snack (🍎)
  - 1:00-3:59 PM → Lunch (🥗)
  - 4:00-6:59 PM → Afternoon Snack (🍌)
  - 7:00-9:59 PM → Dinner (🍽️)
  - 10:00 PM-3:59 AM → Evening Snack (🌙)

AI Analysis Format:
  - Prompt: Instructions for Gemini
  - Input: Photo (base64) or text description
  - Output: Plain text with nutrition table
  - Parse: Regex extracts $$TOTALS: kcal=X, ...$$
  - Result: Structured nutrition data

Authentication Flow:
  1. User visits / or /log
  2. Not authenticated → redirect to /login
  3. Click "Sign in with Discord"
  4. Discord OAuth → confirm identity
  5. Callback to /api/auth/callback/discord
  6. NextAuth creates JWT session with Discord ID
  7. User can now access protected pages

Database Connection:
  - Pool: 5 max concurrent connections
  - SSL: Disabled for railway.internal
  - SSL: Enabled for external databases
  - Queries: Parameterized ($1, $2, etc)
  - No SQL injection vulnerabilities
